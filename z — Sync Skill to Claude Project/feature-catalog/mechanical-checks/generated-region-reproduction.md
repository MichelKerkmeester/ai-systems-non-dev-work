# Generated Region Reproduction
<!-- sk-doc-template: skill_asset_feature-catalog -->

## 1. OVERVIEW

Verifies generated-region markers and detects body drift against the last locked generated bytes.

The check allows hand-authored text outside generated markers while protecting the generated bodies recorded by the previous sync.

---

## 2. HOW IT WORKS

For every `generatedRegions` declaration, the check requires the target file and each named begin and end marker. A missing target or incomplete marker pair produces `REGION_NOT_SCAFFOLDED`.

The check indexes recorded region hashes from `package-lock.json` by target and section. When a locked body exists, it hashes the live extracted body and reports `REGION_DRIFT` if the bytes changed. A region with no prior lock record stays silent until a sync records it.
