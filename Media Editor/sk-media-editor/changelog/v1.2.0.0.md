## [**1.2.0.0**] - 2026-07-18

The Media Editor learns to fall back. When the Imagician MCP server is down for an image operation, or the Video-Audio MCP server is down for a video or audio operation, the CLI runtime now drives Terminal FFmpeg directly for that operation instead of stopping. FFmpeg availability still gates. When the MCP server is down and FFmpeg is also unavailable, the operation blocks and the response says so plainly, so both surfaces down stays a real stop condition rather than a silent degrade. HLS is untouched and still runs on Terminal FFmpeg alone with no fallback. The change moves `SKILL.md` from 1.1.0 to 1.2.0, the advisory kernel `Custom Instructions.md` from 1.0.1 to 1.1.0 aligned to Skill 1.2.0, and the HLS asset from 0.110 to 0.111. It also corrects a fallback statement that ran the wrong direction, naming the Video-Audio MCP server as a fallback for FFmpeg when the runtime need is the reverse.

> Scope: tool verification gate, fallback direction and fallback disclosure. Media rules, format intelligence, MEDIA thinking and the export protocol untouched. HLS mode untouched.

---

### Tool Verification Gate And Routing

The Tool Verification Gate in `SKILL.md` moves from unconditional blocking to MCP-first with a Terminal FFmpeg fallback. Image operations prefer Imagician and fall back to FFmpeg, video and audio operations prefer Video-Audio and fall back to FFmpeg, and HLS stays FFmpeg-only. The gate blocks only when the MCP server and FFmpeg are both unavailable. The MEDIA Flow STEP 2, the Operating Modes table, the ALWAYS, NEVER and ESCALATE rules and the Success Criteria gates all state the same direction. The Smart Router pseudocode gains a `FALLBACK_MAP` and a `resolve_tool_with_fallback` helper that verifies the MCP server first, tries FFmpeg next and raises a blocking error only when neither is available, replacing the old `verify_tool_connection(tool)` blocking call that hardcoded one tool per mode.

&nbsp;

### Fallback Coverage Split

A new Fallback Coverage subsection in `SKILL.md` Integration Points states what Terminal FFmpeg genuinely substitutes for and what it does not, drawn from the real Imagician and Video-Audio tool surfaces rather than an assumed parity. The fallback covers the core deterministic operations: format conversion, resize, crop, compress, rotate and flip for images, plus trim, concatenate, speed, resolution, aspect ratio, codec, bitrate, frame rate and audio extraction for video and audio. Video-Audio is itself an FFmpeg wrapper, so its core operations map directly to FFmpeg command lines. The subsection also names what does not reach parity: Imagician single-call batch arrays become a scripted FFmpeg loop, Sharp quality presets do not map to FFmpeg quality numbers exactly, EXIF orientation auto-correction needs an explicit transpose, and the Video-Audio convenience tools for silence removal, named transitions, overlays, subtitles and b-roll need hand-built filtergraphs. No listed operation is impossible in FFmpeg. The gap is packaging convenience and preset calibration, not raw capability.

&nbsp;

### Fallback Disclosure

A new house rule requires every response that ran through the fallback to state plainly it ran via Terminal FFmpeg rather than the named MCP server, and never to present an FFmpeg result as MCP output. The rule lands in the Tool Verification Gate, in a new ALWAYS entry and in the NEVER list on the skill side. On the advisory side it lands as Critical Rule 9, in the Delivery Protocol post-block instruction and in the Quality Checklist.

&nbsp;

### Advisory Kernel Parity

`Custom Instructions.md` describes the same fallback as guidance so a claude.ai Project reader gets the same mental model as a CLI operator. The Advisory-only truth and Tool reality first rules now name the MCP-first path with the Terminal FFmpeg fallback, the Operating Modes Runtime Tool column shows the fallback per mode, and a new Fallback Path subsection carries the same covered-versus-not-covered split. The Delivery Protocol Tool field states the fallback verification, and the Attestation footer gains a `fallback = [none|Terminal FFmpeg]` field so an advisory answer records which path the CLI runtime would take.

&nbsp;

### Surface Alignment

`AGENTS.md` updates its export strict sequence, its Processing Hierarchy Tool Verification step and its escalation line to the MCP-first, FFmpeg-fallback, block-only-if-both-down direction, keeping the entry wrapper aligned with the gate as one contract. The skill `README.md` fixes the reversed fallback row in Troubleshooting, refreshes the tool-not-connected FAQ, the Tool Verification Gate feature row, the ASCII gate box and the improvement-protocol prose, and moves to 1.2.0.0. The HLS asset reframes its MCP-versus-terminal note so it no longer casts Terminal FFmpeg as HLS-only, and it records that FFmpeg is also the fallback for single-format conversions and editing when the MCP server is down. `SYNC.md` updates its overview and Required Parity gate language to the fallback direction and re-pins the Skill, Custom Instructions and mirror versions in the inventory.
