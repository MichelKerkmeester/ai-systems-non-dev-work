## [**1.1.1.0**] - 2026-07-17

Correction and alignment release. `claude project/Custom Instructions.md` moves from kernel `v1.0.0` to `v1.0.1`, closing four contract gaps against `SKILL.md` v1.1.0. The Guidance Block Tool field now names `list_images`, `health_check` and `ffmpeg -version` as the verification command per mode. The Feature Realism rule now states the 100MB MCP and 5GB HLS hard limits. The Smart Routing Logic section now states that explicit command tokens override natural-language scoring. The Attestation footer now states plainly that execution, verification and save did not occur. `claude project/README.md` closes three matching operational gaps in the same pass, covered below. `SKILL.md` itself is unchanged, since every contract detail these fixes require already existed at source. Only the derived kernel and README needed to catch up.

> Scope: kernel and README correction only. Skill routing, tool bindings and media rules untouched.

### Custom Instructions Corrections

Four kernel gaps close in this pass. Section 7's Guidance Block Tool field previously read `Imagician, Video-Audio or FFmpeg, plus a verification command or check` with no command named. It now spells out `list_images` for Imagician, `health_check` for Video-Audio and `ffmpeg -version` for FFmpeg per mode, matching `SKILL.md`'s Tool Verification Gate. Section 2's Feature Realism rule previously flagged limits by category only. It now states the two hard numbers, over 100MB for MCP operations and over 5GB for HLS, matching `SKILL.md`'s NEVER rule 3. Section 5's Smart Routing Logic previously opened straight into the routing table with no precedence statement. It now states that explicit command tokens override natural-language mode scoring, matching `SKILL.md`'s Routing Checks. Section 7's Attestation footer previously carried only `execution = advisory-only`. It now states plainly that execution, verification and save each did not occur, matching the advisory-only attestation contract.

### README Corrections

Three operational gaps close alongside the kernel. The Set Up smoke test previously exercised four of six modes. It now adds `$repair` and `$interactive` to the existing four command tests plus the ambiguous-request case. The Guidance Block validation step previously checked only the attestation footer and the advisory-only statement. It now requires all six Guidance Block fields, Deliverable, Tool, Recipe, Settings, Expected result and Attestation, plus the two-to-three sentence post-block instructions. The unconditional "keep filenames unchanged" rule is replaced with the version-bump-and-rename rule already governing the Change Checklist. A final Change Checklist step now requires every `SYNC.md` section 7 drift check to pass before release.
