## [**1.0.0.0**] - 2026-06-15

The Media Editor's dual-runtime packaging lands for the first time. A CLI-facing `skill/` folder pairs with a chat-advisory `claude project/` mirror, and `SKILL.md` becomes the one brain, replacing an older routing model where a separate always-loaded document carried that role. The system's first git-tracked appearance in this repository predates this release: on 2026-04-10 it lived at `MCP Agents/Media Editor/` with only the `knowledge base/` and `mcp servers/` pairing described below, no `skill/` or `claude project/` split. That grouped-path version was deleted outright on 2026-05-21 as part of a separate restructuring pass, and this 2026-06-15 commit is the first to add content back, under the new flat `Media Editor/` path and already in the dual-runtime shape. Git records a delete and a later add rather than a rename between the two. `SKILL.md`'s frontmatter carries version 1.0.0, the first system-level version number the Media Editor has ever had, since the pre-2026-06-15 architecture used only per-document versions (Prompt v0.240, MEDIA Framework v0.233, Interactive Intelligence v0.220, the two MCP integration specs at v0.211 and v0.212, HLS at v0.110) with no single release number for the system as a whole. Changelog version records for this system begin here.

> Scope: dual-runtime packaging rollout and a same-day Human Voice re-freeze. Media rules, tool bindings and the export protocol carry over unchanged from the prior architecture.

---

### Dual-Runtime Packaging

`skill/` becomes the CLI source of truth, holding `SKILL.md` plus `assets/hls_video_conversion.md` and five `references/` files (`human_voice_rules.md`, `interactive_intelligence.md`, `media_framework.md`, `mcp_imagician.md`, `mcp_video_audio.md`), all underscore-named at this point. `claude project/` becomes the chat-advisory mirror, with `Custom Instructions.md`, a `README.md` and a `knowledge/` folder holding seven files, including the first `Media Editor - System - Skill - v1.0.0.md`. The move retires an older routing model where a separate always-loaded `Media Editor - System - Prompt - v0.240.md` document carried the routing logic and the prior AGENTS.md pointed to it as "the System Prompt" rather than embedding routing directly. `SKILL.md` absorbs that role, and the command set it inherits (`$image`, `$video`, `$audio`, `$hls`, `$repair`, `$interactive`) stays the same set the old System Prompt already routed.

&nbsp;

### Legacy Knowledge Base Retained

The pre-conversion `knowledge base/` folder is not deleted, only demoted. It keeps its own `integrations/`, `reference/` and `system/` subfolders and six files, including `Media Editor - System - Prompt - v0.240.md`, which has no equivalent among the seven `claude project/knowledge/` mirrors. That file is the retired always-loaded system prompt the dual-runtime model replaces, kept for manual comparison instead of being wired into either runtime.

&nbsp;

### Tool Layer Asymmetry

The two MCP servers package differently from the start. `mcp servers/imagician/` vendors a prebuilt `dist/index.js` alongside its own `package.json`, and its Dockerfile installs dependencies and copies the compiled bundle directly. `mcp servers/video-audio/` carries no `dist/` folder and no `package.json`. Its Dockerfile instead clones `misbahsy/video-audio-mcp` from GitHub at build time and installs Python requirements from that clone. Same tool layer, two different build strategies, present from the system's earliest tracked commit under this new path.

&nbsp;

### Same-Day Human Voice Re-Freeze

A second commit later that same morning converts the Human Voice mirror to the sk-doc reference template and refreshes `claude project/README.md`, touching only those two files. 106 lines change in the Human Voice mirror alone. No Skill version bump accompanies it. This re-freeze produces the `f8980baed64d` checksum lineage that later README manifests still carry for that file.
