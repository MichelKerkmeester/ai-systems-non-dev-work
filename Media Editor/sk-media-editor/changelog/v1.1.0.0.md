## [**1.1.0.0**] - 2026-07-16

Packaging alignment release. The skill folder renamed from `skill/` to `sk-media-editor/` during the nine-system alignment wave, then to `mcp-media-editor/` on 2026-07-17 by owner decision since this system fronts MCP servers rather than authoring documents outright. Git history for this folder shows only the final hop as a tracked rename, from `skill/SKILL.md` straight to `mcp-media-editor/SKILL.md`, so the intermediate `sk-media-editor/` name lived in the working tree and never reached its own commit. The same pass hyphenates `INSTALL_GUIDE.md` to `install-guide.md` and both `VOLUME_REFERENCE.md` files under `mcp servers/` to `volume-reference.md`, salvages the old 434-line root `README.md` into `z_legacy/Media Editor - System - User Guide - v0.212.md`, and leaves a symlink at `README.md` pointing to the newly authored `mcp-media-editor/README.md`. `SKILL.md` gets its first structural rewrite since the June baseline, its frontmatter version moving from 1.0.0 to 1.1.0, and the release closes out an unversioned 2026-07-13 standardization pass across the `claude project/` documents, covered below.

> Scope: packaging and structure only. MCP tool bindings, media rules and mode canon untouched.

---

### Folder Identity And Renames

The `skill/` to `mcp-media-editor/` rename is the only folder move this repository's git history records for this system, even though the owner decision named two hops. The intermediate `sk-media-editor/` name existed only in the working tree before the final name replaced it, so no commit ever carries that string as a live path. The `knowledge base/` and `mcp servers/` directories, both untouched by the rename itself, were swept for stale references to the old skill path. 82 references across the system were repointed, and a post-rename check across the folder tree returned zero residuals pointing at the retired `skill/`, `sk-media-editor/` or `INSTALL_GUIDE.md` names.

&nbsp;

### Package Surface

`SKILL.md` was rewritten to the create-skill standard behind a characterization harness, and the frontmatter identity `name: media-editor` stays the invocation contract across both folder renames. Advisor metadata lands for the first time: `description.json` (53 lines, carrying the standalone-skill description, a 29-entry keyword list and ten trigger examples) and `graph-metadata.json` (149 lines, carrying the skill's edges, domains, intent signals and a derived causal summary for the code graph). The new `mcp-media-editor/README.md` (390 lines) is freshly authored rather than mechanically ported from the old root file: the retired 434-line README carried no mention of Repair Mode at all, while the new README documents Repair Mode's auto-detect behavior, its keyword triggers and its connection-check fallback in detail. `SYNC.md` grows from a flat 67-line note into a 144-line manifest with eight numbered sections following the sk-doc section standard, from Overview through Related Documents.

&nbsp;

### SKILL.md Consolidation

`SKILL.md` itself shrinks from 394 lines to 369 despite the added detail, because three duplications collapse into one telling. The old Section 2 `Phase Detection` ASCII diagram is removed outright: it repeated the same six-step sequence the Section 3 MEDIA Flow already states, just numbered as `STEP 0-2` plus `Phase 1-3` instead of `STEP 1-6`. Six reference and asset filenames convert from underscore to hyphen (`media_framework.md`, `human_voice_rules.md`, `mcp_imagician.md`, `mcp_video_audio.md`, `hls_video_conversion.md`, `interactive_intelligence.md`), matching the folder-wide hyphenation. The router's `_guard_in_skill` function switches from `.resolve()` to `.absolute()`, with an added comment explaining that `.resolve()` would dereference the relative `human-voice-rules.md` symlink and escape the skill root, failing the guard for an ALWAYS-loaded resource. The comment cites the identical fence already applied on Product Owner's shared-symlink shape. Section 7 folds its old `Tool Layer` and `Knowledge Base Dependencies` subsections into one paragraph that points back to Section 3 and Section 5 instead of repeating them, and the standalone `Section 8 References And Related Resources` disappears into Section 5, taking the document from eight top-level sections to seven. A phantom `brand` symlink mention (the old Resource Domains line named a `brand` global this skill never actually loads) is removed at the same time, leaving only `human-voice-rules` in that line.
