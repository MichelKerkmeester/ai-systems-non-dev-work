## [**1.4.0.1**] - 2026-07-17

Correction and alignment release. The `1.4.0.0` release synthesized `Custom Instructions.md` against Skill `1.4.0` but the kernel had drifted from its own source mirror in eight places: a runtime contradiction over save-and-read-back delivery, two missing energy and confidence rules, a non-blocking perspective floor, a thin regression matrix, two stale version pins and two stale cross-reference names in the shared references. `SKILL.md` itself stays pinned at `1.4.0`; every change below is a kernel, README or reference correction that brings the package back into alignment with the Skill it already claimed to match.

> Scope: eight kernel and reference corrections plus their version bumps, re-mirror and checksum refresh. Router fixture suite ran 43/43 before and after every kernel edit. No routing, template or delivery behavior changed beyond the eight items below.

---

### Deliverable Block Precedence Stated

`Custom Instructions.md`'s Export-Equivalent Paths section named the runtime's `Path:`/`Saved:`/`Verified:` restriction but never said outright that this rule overrides the retrieved System Skill mirror's own filesystem save-and-read-back instructions. The section now states that the Deliverable Block protocol overrides those mirror instructions, so the contradiction resolves inside the kernel instead of staying implicit.

&nbsp;

### Raw Energy And Skip Depth Added

The kernel's Processing list named only Standard, Quick, Deep and Interactive, with no Raw energy and no `skip depth` trigger anywhere in the file. A `DEPTH Raw` line joined the Processing list and a new Routing and Energy rule states that `skip depth` is the explicit trigger for Raw energy and that `$quick` never selects it, matching `SKILL.md`'s energy-level definition.

&nbsp;

### Confidence Thresholds Made Explicit

The kernel routed by confidence without ever naming a threshold. A new Routing and Energy line states the four bands verbatim from `SKILL.md`: HIGH `>= 0.85` direct, MEDIUM `>= 0.60` confirmed, LOW `>= 0.40` suggested through Interactive Mode, FALLBACK `< 0.40` one comprehensive question.

&nbsp;

### Perspective Floors Made Blocking

The Quality Gate table carried the six dimension floors but no perspective-count rule, while `SKILL.md` requires Standard to clear 3 of 5 canonical perspectives and Deep to clear all 5 as a blocking floor. The Quality Gate now states both counts as floors that block delivery exactly like the dimension floors, and the closing Quality Checklist gained Story alongside Task, Bug and Doc, matching `SKILL.md`'s five-intent list.

&nbsp;

### ClickUp Consent And Read-Back Generalized

Two consent lines and the ClickUp Connector Delivery section named ClickUp specifically, while `SKILL.md`'s own NEVER rule already generalizes the same consent to "ClickUp or another external system." Both kernel lines now match. The Connector Delivery section also gained the task read-back step, `include_markdown_description=true` confirmed against the pushed body before delivery is reported complete, matching `SKILL.md`'s ClickUp Transport table.

&nbsp;

### Stale Pin And Inventory Wording Corrected

`claude project/README.md`'s Smoke Matrix cited pinned `v0.302` and `v0.201` Task and Bug references against live `v0.303` and (post-bump) `v0.203` files, and `Custom Instructions.md` described "thirteen uploaded Project Knowledge mirrors" against a live count of thirty. Both now read the current versions and the accurate thirteen-core-plus-seventeen-example split.

&nbsp;

### Stale Cross-References Repaired At Source

`bug-mode.md` pointed at a nonexistent "Owner - Templates - Task Mode" and named its Bug Report Template link with a directory-relative path that only resolves inside the skill tree, not the flat Project Knowledge layout. `depth-framework.md` pointed at a nonexistent "Owner - Rules - Human Voice.md" in two places. All three now use the plain in-house names this repository's own files already use elsewhere (`Templates - Task Mode`, `Assets - Bug Report Template`, `Human Voice Rules`), which resolve identically in both the skill tree and the flat mirror layout. `bug-mode.md` moved `v0.202` to `v0.203` and `depth-framework.md` moved `v0.201` to `v0.202`; both mirrors re-derived with `cp -L` and the superseded mirror filenames were removed.
