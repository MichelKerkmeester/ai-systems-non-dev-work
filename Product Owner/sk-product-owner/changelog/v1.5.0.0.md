## [**1.5.0.0**] - 2026-07-22

Story Mode house-format release. Story Mode now drafts in the Barter ClickUp house format by default, the shape the team already writes, and the earlier ceremony (a Connextra line inside every requirement, a Rule block, a Which-means-that block, Definition of Ready/Done gates) becomes optional enrichment a writer pulls in only when a story earns it. The change came from an audit of five real house stories against the old template: the template prescribed a heavier shape than the team uses, so the template moved to the team rather than the other way around.

> Scope: a story-templates rewrite, story-mode logic alignment, three reshaped examples plus one new maximal reference example, four SKILL.md grammar carve-outs, the kernel and README equivalents, and the full re-mirror and checksum refresh. No Task, Bug, Doc or Interactive behavior changed. Router token rules, tie-breaks and Quick-energy handling are untouched.

---

### House Format Is Now The Default Story Shape

`assets/story-templates.md` (v0.102 to v0.200) replaced the three separate tier scaffolds with one house shape sized to the requirement count: the italic blockquote preamble, an `### About` block carrying the User Story promise plus Problem and Solution, `### Requirements` where each requirement has a short prose lead and a `**Checklist**` of buildable `- [ ]` items, and `### Acceptance criteria` written as numbered `1.` Given/When/Then blocks each closed with `- [ ] _Mark as done, if the criteria are met_`. Section anchors sit at H3 because ClickUp renders H2 very large. The H1 slot is documented as `Story - {Platform/Area} - {Feature} - {Name}` with an open set of house codes (`BO`, `BE`, `FE`, or a named area) plus an optional version tag.

&nbsp;

### Heavier Machinery Moved To Optional Enrichments

The per-requirement Connextra value line, the exact Rule block, the Which-means-that implications, the Definition of Ready and Definition of Done gates, the shared-mechanism section and the `← PRIO` marker are now a catalog of optional enrichments in section 4 of the templates, added only when a story earns them. A Simple story typically uses none; a Complex story typically uses several.

&nbsp;

### Story Mode Logic Aligned

`references/story-mode.md` (v0.103 to v0.200) restated tiers as size only, with every tier rendering the same house shape. Acceptance criteria are numbered Given/When/Then blocks rather than named `**Scenario:**` blocks, which retired the create-time scenario-count export gate: a story is validated by coverage and requirement-tracing, not by a heading count. A note routes research and feasibility spikes to Task or Doc Mode, or keeps them on the story surface with acceptance that asserts the documented decision. The refinement rule now states plainly that create-time shape checks do not fire on a source-preserving refinement, resolving the earlier collision between the export gate and refinement preservation.

&nbsp;

### Examples Reshaped And A Maximal Reference Added

The three worked examples (Simple, Medium, Complex, each v0.100 to v0.200) were rewritten in the house shape, sized to show the range: Simple uses no enrichments, Medium adds a Documentation link and one Rule block, Complex adds the shared-mechanism section, a `← PRIO`, per-requirement value lines and a Definition of Done. A new `story-example-complete.md` (v0.100) is the maximal reference: every section and every optional enrichment populated in one story, so an author can see the full menu at once.

&nbsp;

### SKILL.md Grammar Carve-Outs

`SKILL.md` (v1.4.0 to v1.5.0) gained four surgical carve-outs so the router's grammar contract recognizes the Story house format distinct from Doc: Story artifacts use `- [ ]` checklists (Doc keeps `*   [ ]`), H3 section anchors (Doc keeps H2), numbered acceptance criteria with a Mark-as-done checkbox, and the blockquote preamble. The Mode Selection note and the Story blocking gate now name the house format. No routing, token or energy rule changed.

&nbsp;

### Kernel And README Equivalents

`Custom Instructions.md` (v1.3.0 to v1.4.0, aligned to Skill v1.5.0) took the equivalent kernel edits: the Story tier guard now counts acceptance criteria rather than scenarios and drops the hard blocking-failure framing, a new-Story grammar rule joined the ClickUp-format section, and the Story routing row names the house format and optional gates. `claude project/README.md` updated its source-to-mirror map, its aligned-Skill version and its mirror counts.

&nbsp;

### Manifest And Version Bumps

`claude-project.sync.json` bumped the kernel to v1.4.0 aligned to Skill v1.5.0, moved the Skill, Story Mode, Story Templates and three Story example mirrors to their new versions, added the Complete Reference example mirror, raised `expectedKnowledgeCount` from 30 to 31, and retired the six superseded mirror names. The compiler re-rendered all mirrors, the SYNC.md inventory region and the README checksum region against final bytes.
