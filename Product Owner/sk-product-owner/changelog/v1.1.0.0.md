## [**1.1.0.0**] - 2026-06-15

Skill identity formation release. Product Owner's dual-runtime packaging began five days earlier on 2026-06-10, when the system-wide `AI systems: skill + claude-project packaging model across all 10 systems` commit gave Product Owner its first `skill/` plus `claude project/` split, and `skill/SKILL.md` was already pinned at `1.0.0` beside the legacy `Product Owner - System - Prompt - v0.997.md` identity file (see the 1.0.0.0 entry). On this date the flatten-layout commits deleted the Prompt file, made SKILL.md the sole identity and bumped it to `1.1.0`. That version number holds for the rest of this entry's span: it covers the 2026-06-10 packaging origin, the 2026-06-15 SKILL.md birth and Human Voice Rules re-freeze and a 2026-07-13 Claude Project kernel milestone, because none of those later dates changed the pinned `1.1.0` skill version. The version next moves in the `1.4.0.0` release.

> Scope: skill identity formation and dual-runtime packaging. Task, Bug and Interactive were the only intents. Doc and Story did not exist yet.

---

### Dual-Runtime Packaging Origin (2026-06-10)

The system-wide packaging commit gave Product Owner its first `skill/` and `claude project/` split. `AGENTS.md` grew by 235 lines into the CLI bootstrap and enforcement wrapper, `SYNC.md` and `claude project/Custom Instructions.md` were both created, and `claude project/README.md` landed as the upload manifest. The CLI identity file at this point was `Product Owner - System - Prompt - v0.997.md` at 478 lines, paired with `Interactive Mode v0.401`, `Bug Mode v0.200`, `Task Mode v0.300`, `DEPTH Framework v0.201` and `Human Voice Rules EN v0.210`. The same commit removed a large block of stale product context (old App Store descriptions, superseded DS Variables exports, retired Feed v2 and v3 documentation) that had accumulated ahead of a full-service migration.

&nbsp;

### SKILL.md Born At v1.1.0 (2026-06-15)

The flatten-layout commit deleted the 534-line `System - Prompt - v0.997.md` and created `skill/SKILL.md` at 721 lines, version `1.1.0`: the first genuine single-brain skill file for Product Owner, replacing the older prompt-file concept. The same commit split three template files out of the mode references into dedicated assets for the first time, `skill/assets/task_templates.md` (262 lines), `skill/assets/bug_report_template.md` (120 lines) and `skill/assets/interactive_response_templates.md` (92 lines), and bumped `Interactive Mode` to `v0.402`, `Bug Mode` to `v0.201` and `Task Mode` to `v0.301` to match. The commit message names this a fleet-wide normalization pass that put `## 1. OVERVIEW` at the top of every real skill reference and asset.

&nbsp;

### Human Voice Rules Re-Freeze (2026-06-15)

A same-day follow-up commit converted the English and Dutch Human Voice Rules to the sk-doc reference template (five-field frontmatter, a one to two sentence intro, `## 1. OVERVIEW`) and re-froze the shared scoring baseline. The commit message records the HVR checksum moving from `3f092d99b55b` to `f8980baed64d` while stating the banned-word, blocker and deduction content stayed byte-unchanged: the re-freeze touched structure and cross-reference numbering, not scoring rules.

&nbsp;

### Claude Project Kernel Reaches v1.0.0 (2026-07-13)

A later system-wide standardization commit bumped `claude project/Custom Instructions.md` from `v0.998` to `v1.0.0` while `skill/SKILL.md` stayed pinned at `1.1.0`. The same commit rebuilt `claude project/README.md` around a new `Paired-Version + Checksum Table` that lists each knowledge mirror's version and sha256-16 checksum in one place, replacing the plain `## Upload` list the file used before. That table shape is the direct ancestor of the thirty-row inventory the `1.4.0.0` release later expands to cover Doc and Story mirrors.
