## [**1.2.0.0**] - 2026-07-16

Packaging alignment release that brings Prompt Improver's on-disk layout in line with the sk-doc packaging convention already rolling out across the rest of the Barter AI systems fleet. The same rename-and-hyphenate pattern lands in Copywriter, Deal Templates, Blog Posts, both LinkedIn systems, both Sales systems and Media Editor. The skill folder renames from `skill/` to `sk-prompt-improver/`, all 13 underscored reference and asset filenames convert to hyphenated form, SKILL.md is rewritten against the create-skill standard behind a characterization harness, the 927-line root README is salvaged into a new descriptive skill README with a symlink left in its place, SYNC.md is restructured to the numbered sk-doc section standard and a pair of advisor metadata files ship for the first time. A closing audit pass catches and fixes leftover drift between the mirrored surfaces. The version pin moves from 1.1.2 to 1.2.0 across SKILL.md, Custom Instructions.md and every paired knowledge mirror.

> Scope: packaging and structure only. CLEAR scoring and framework selection untouched.

---

### Folder Rename And Hyphenation

The source-of-truth folder renames from `skill/` to `sk-prompt-improver/`. Inside it, 13 underscored filenames convert to hyphenated form: the 6 references (`depth-framework.md`, `interactive-mode.md`, `patterns-evaluation.md`, `visual-mode.md`, `image-mode.md`, `video-mode.md`) and the 7 assets (`framework-pattern-library.md`, `format-guide-markdown.md`, `format-guide-json.md`, `format-guide-yaml.md`, `visual-mode-library.md`, `image-mode-library.md`, `video-mode-library.md`). Every cross-reference inside SKILL.md, AGENTS.md and the reference files themselves updates to the new hyphenated paths in the same pass, and the versioned `claude project/knowledge/` mirror filenames stay display-named rather than hyphenated, only their paired source paths change.

&nbsp;

### SKILL.md Rewrite

SKILL.md rewrites against the create-skill standard behind a characterization harness, dropping from 749 to 663 lines. It keeps its section shape but folds the old standalone "REFERENCES AND RELATED RESOURCES" section into a new "Dynamic Discovery" subsection under section 5 REFERENCES, taking the file from eight top-level sections to seven. The smart-router pseudocode block sheds dead weight that the live `route_prompt_improver_resources` path never called: the `MODE_CONFIDENCE`, `PRELOAD_GROUPS`, `MODE_PATTERNS`, `UI_PLATFORM_PATTERNS`, `IMAGE_PATTERNS`, `VIDEO_PATTERNS` and `FRAMEWORKS` dictionaries, the unused `HIGH`, `MEDIUM` and `FALLBACK` confidence tiers and the `detect_mode`, `detect_framework` and `detect_complexity` helper functions all go. The frontmatter version moves from 1.1.2 to 1.2.0.

&nbsp;

### README Salvage And Legacy Preservation

The 927-line root `README.md`, still headed "Prompt Engineering Assistant - User Guide v0.200," is retired from the root and preserved verbatim as `z_legacy/Prompt - Root README (superseded) - v0.980.md`. A new 466-line descriptive skill README lands at `sk-prompt-improver/README.md`, and the root `README.md` becomes a symlink pointing at it, so anything reading the root-level file still lands on current content with no second copy left to drift out of sync.

&nbsp;

### SYNC.md Restructure

SYNC.md grows from a 43-line, five-section informal manifest into a 153-line, eight-section document numbered to the sk-doc standard: OVERVIEW, WHEN TO SYNC, MANUAL-LOAD CONTRACT, SOURCE-TO-MIRROR INVENTORY, SYNC WORKFLOW, REQUIRED PARITY, DRIFT CHECKS and RELATED DOCUMENTS. The new REQUIRED PARITY section spells out, for the first time in this file, the exact ten routed intents, command aliases, DEPTH energy and phase pairings and scoring floors both packagings must agree on, and DRIFT CHECKS closes with a runnable manual check sequence in place of the old prose description.

&nbsp;

### Advisor Metadata

Two new files ship for the first time: `description.json`, carrying the skill id, a 24-entry keyword list and ten trigger examples for advisor routing, and `graph-metadata.json`, a schema-versioned sibling-edge map naming `sk-prompt` at weight 0.4 as the prompt-craft peer and `sk-doc` at weight 0.2 as the documentation-packaging peer. Neither file existed before this release.

&nbsp;

### Verification And Audit

14 knowledge mirrors come back byte-identical against their `sk-prompt-improver/` sources, and all 15 rows of the Paired-Version and Checksum Table, the 14 knowledge mirrors plus Custom Instructions itself, get their sha256-16 values recomputed against final bytes. The closing audit catches and fixes four findings. Two are directly visible in the diff: AGENTS.md's resource dependency diagram was missing entries for the three mode-library assets (`visual-mode-library.md`, `image-mode-library.md`, `video-mode-library.md`) and gains them, and `claude project/Custom Instructions.md` carried an orphaned "Human Voice Rules pass across all modes" line with no counterpart anywhere in SKILL.md, which the pass replaces with SKILL.md's own long-standing creative-follow-up rule inviting the user to share the generated result for one more refinement pass.
