## [**1.2.1.0**] - 2026-07-17

Correction and alignment release closing fourteen of the fifteen confirmed kernel, source and README gaps found against the authoritative Skill. The Custom Instructions kernel moves from 1.2.0 to 1.2.1, and SKILL.md moves past 1.2.0 to 1.2.1 for the new Claude Projects delivery override. Patterns and Evaluation, Visual Mode, Image Mode and Video Mode bump for a package-relative link fix, and both non-Markdown format guides bump for the same delivery override that lands in SKILL.md. One further finding, adding RAG frontmatter fields to SKILL.md, is carried as an amendment candidate rather than applied, because the fleet-wide convention across all ten systems keeps that frontmatter to name, description, allowed-tools and version.

> Scope: kernel correction, source delivery-override and link fixes, README and SYNC fact updates. Fourteen of fifteen confirmed findings land. One is escalated rather than forced.

---

### Command Precedence And Raw Trigger Correction

The kernel now states that explicit `$` commands override conflicting natural-language signals, and that a clarifying question is required when multiple commands conflict, matching the Skill's router authority and its command-conflict escalation rule. The Raw path entry in the router diagram drops the natural-language bypass triggers `"clean this"` and `"no scoring"`, leaving `$raw` as the only way into Raw.

&nbsp;

### Deep Framework Default And MagicPath Sub-Gates

The Deep path no longer names a fixed TIDD-EC/CRAFT default. It now reads complexity-matched framework selection, consistent with the Skill's own Deep mapping and the kernel's lightest-fitting-framework rule. The EVOKE threshold rule gains the MagicPath Kinetic (8+/13), Visual (8+/12) and combined Kinetic+Visual (18+/25) sub-gates alongside the existing 42+/50 total.

&nbsp;

### Deliverable Block Header And Format-Lock Boundary

The Deliverable Block header gains a Complexity field, matching the Markdown format guide's own header contract, and the Block-contents rule now names complexity alongside mode, framework, score and attestation. A new line defines the header and attestation as delivery metadata that frame the payload, sitting outside the JSON/YAML format lock, which applies only to the enhanced prompt content between them. The post-block chat contract and its matching rule gain a token-overhead line for JSON and YAML deliverables.

&nbsp;

### Claude Projects Delivery Override

`SKILL.md`, `assets/format-guide-json.md` and `assets/format-guide-yaml.md` each gain an explicit Claude Projects delivery branch. `SKILL.md` states plainly that claude.ai Projects cannot write files and that the Deliverable Block substitutes for the saved file, and both format guides add a third Delivery Methods entry making the same point next to their existing Claude Desktop/IDE and CLI/Agent Mode entries. This stops retrieved knowledge from reintroducing CLI-only file-delivery language on the Claude surface, without dropping any of the CLI runtime's own export-first requirement.

&nbsp;

### Package-Relative Link Cleanup

`references/patterns-evaluation.md`, `references/visual-mode.md`, `references/image-mode.md` and `references/video-mode.md` drop every `../assets/...` package-relative link, seventeen occurrences in total, converting each to a plain-text asset mention. `claude project/` has no `references/` or `assets/` subdirectory, so the old links never resolved there.

&nbsp;

`claude project/README.md`'s parity claim, Set Up smoke-test line, Change Checklist and Known Notes all update.
